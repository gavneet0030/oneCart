<!-- ================= PURPOSE ================= -->
<h2>Purpose</h2>

<p>
  The primary purpose of OneCart is to design, develop, and demonstrate a
  complete online e-commerce web application that closely reflects the
  functionality, architecture, and workflow of real-world digital shopping
  platforms.
</p>

<p>
  OneCart aims to provide a structured understanding of how modern web
  applications handle authentication, product catalog management, shopping
  cart operations, order processing, and administrative control through a
  secure and scalable client–server architecture.
</p>

<h3>Educational Purpose</h3>
<ul>
  <li>Understand full-stack web development using layered architecture</li>
  <li>Apply software engineering concepts in a real project</li>
  <li>Implement secure authentication and authorization</li>
  <li>Practice RESTful API and database integration</li>
</ul>

<h3>Practical & Technical Purpose</h3>
<ul>
  <li>Build a scalable and modular e-commerce system</li>
  <li>Separate frontend, backend, and database responsibilities</li>
  <li>Manage real-time user interactions and data persistence</li>
  <li>Support role-based access for users and administrators</li>
</ul>

<h3>System-Oriented Purpose</h3>
<ul>
  <li>Ensure secure data flow between client and server</li>
  <li>Maintain data consistency across cart and order lifecycle</li>
  <li>Enable administrative control over products and orders</li>
  <li>Simulate real-world online shopping workflows</li>
</ul>

<!-- ================= PROJECT OVERVIEW ================= -->
<h2>Project Overview</h2>

<p>
  OneCart is a comprehensive full-stack e-commerce web application that allows
  users to browse products, manage a shopping cart, place orders, and track
  purchase history through a secure and interactive interface.
</p>

<p>
  The application is divided into three major components:
  frontend user interface, backend API server, and database layer. These
  components communicate through RESTful APIs, ensuring separation of concerns
  and easier maintenance.
</p>

<h3>User-Oriented Capabilities</h3>
<ul>
  <li>Secure account creation and login</li>
  <li>Product browsing with detailed views</li>
  <li>Cart management (add, update, remove)</li>
  <li>Order placement and history tracking</li>
  <li>User profile management</li>
</ul>

<h3>Administrator-Oriented Capabilities</h3>
<ul>
  <li>Admin authentication and access control</li>
  <li>Product creation, update, and deletion</li>
  <li>Inventory and catalog management</li>
  <li>Order monitoring and status updates</li>
</ul>

<h3>Architectural Overview</h3>
<ul>
  <li>Client–server based architecture</li>
  <li>RESTful frontend-backend communication</li>
  <li>Role-based access control</li>
  <li>Centralized database storage</li>
  <li>Modular and extensible design</li>
</ul>
